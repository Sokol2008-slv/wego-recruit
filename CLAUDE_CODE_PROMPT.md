# ПРОМПТ ДЛЯ CLAUDE CODE — Рекрутинговая платформа

## Контекст проекта

Мы строим рекрутинговую платформу для трудоустройства рабочих в Польше и Германии.
Платформа обслуживает ТРИ типа пользователей (сейчас делаем первые два):

1. **Синие воротнички** — разнорабочие, заводы, склады, строительство (ПРИОРИТЕТ СЕЙЧАС)
2. **Офисные сотрудники** — HR-специалисты, менеджеры (добавим позже)
3. **Работодатели** — компании, которым нужны люди (от 50 до 500+ человек)

Проект планируется на тысячи пользователей. Сразу закладываем правильную архитектуру.

---

## Стек

- **Next.js 15** + TypeScript + Tailwind CSS
- **Supabase** — база данных (PostgreSQL), авторизация, хранилище
- Деплой на **Vercel**
- Шрифты: Bebas Neue (заголовки) + Manrope (текст), Cyrillic

В ZIP уже есть базовая структура проекта с настроенным package.json, tsconfig, tailwind, типами и Supabase клиентом.

## Что нужно построить

### 1. Главная страница `/`

Два больших блока-карточки:
- **«Ищу работу»** → ведёт на `/worker`
- **«Ищу сотрудников»** → ведёт на `/employer`

Внизу: полосы с отраслями (заводы, склады, строительство, фермы, пароходы).
Навигация: логотип РАБОТАПРО (ПРО — оранжевый).

---

### 2. Поток соискателя `/worker`

**Шаг 1 — Выбор категории** (пока только одна активна):
- Синие воротнички (активно)
- Офисные специалисты (заглушка, "скоро")

**Шаги анкеты** (пошагово, один вопрос на экране, прогресс-бар сверху):
1. Возраст: До 25 / 25–35 / 35–45 / 45–55 / 55+
2. Пол: Мужской / Женский / Семейная пара
3. Нужно ли жильё: Да / Нет / Не важно
4. Страна: Польша / Германия / Любая
5. Гражданство: Украина / Беларусь / Молдова / Латвия / Литва / Грузия / Армения / Казахстан / Другое
6. Когда готов выехать: Через 3 дня / Через неделю / Через 2 недели / Через месяц

**Шаг — Тест** (после анкеты, перед контактами):

Это короткий психологический тест (5 вопросов). Заглушка — реальные вопросы добавим позже.
Пример вопросов сейчас:
- "Если работа срочная и нужно выехать через 3 дня — ты готов?" → Да / Скорее да / Скорее нет / Нет
- "Ты готов работать в ночные смены?" → Да / Иногда / Нет
- "Как ты относишься к физическому труду?" → Люблю / Нормально / Не очень / Не хочу
- "Есть ли у тебя опыт работы за рубежом?" → Да, есть / Нет, но хочу / Не уверен
- "Документы в порядке (паспорт, виза если нужна)?" → Да, всё готово / В процессе / Нет

После теста — подсчёт score (0–100), показать результат: "Отличный кандидат" / "Хороший кандидат" / "Нужна подготовка".

**Последний шаг — Контакты:**
- Имя
- Телефон / WhatsApp
- Telegram (опционально)

По кнопке «Отправить» — сохранить всё в Supabase (таблица `candidates`) + показать страницу с вакансиями.

---

### 3. Страница вакансий `/vacancies`

После заполнения анкеты соискатель попадает сюда.
Фильтр по стране (Польша / Германия / Все).

Карточка вакансии содержит:
- Название, компания, город, страна (флаг)
- Зарплата крупно (оранжевый)
- Количество мест
- Теги (специальность, пол, возраст)
- Бейдж "🏠 Жильё" если есть
- Дата выезда
- Кнопка "Откликнуться" → модалка подтверждения

Вакансии пока хардкодом (массив в коде), позже подключим из Supabase.

---

### 4. Поток работодателя `/employer`

Одна страница с формой (не пошагово — всё сразу):

**Секция "О компании":**
- Название компании *
- Страна (Польша / Германия / Чехия / Нидерланды / Другая) *

**Секция "Требования к персоналу":**
- Специальность (select: разнорабочие, склад, строители, сварщики, водители, завод, сельхоз, другое) *
- Количество человек * (число)
- Дата начала работы * (date picker)
- Жильё (select: предоставляем / сами ищут / обсудим)

**Секция "Контакты":**
- Контактное лицо *
- Телефон *
- Примечания (textarea, не обязательно)

По отправке → сохранить в Supabase (таблица `employer_requests`) → страница подтверждения.

---

### 5. Supabase — схема таблиц

Создай файл `lib/supabase-schema.sql` с SQL для создания таблиц:

```sql
-- Кандидаты
create table candidates (
  id uuid default gen_random_uuid() primary key,
  created_at timestamptz default now(),
  name text not null,
  phone text not null,
  telegram text,
  age_range text,
  gender text,
  housing_needed boolean,
  country text,
  citizenship text,
  departure text,
  test_score integer,
  test_answers jsonb,
  source text default 'direct',
  status text default 'new',
  category text default 'blue_collar'
);

-- Заявки работодателей
create table employer_requests (
  id uuid default gen_random_uuid() primary key,
  created_at timestamptz default now(),
  company text not null,
  country text not null,
  specialty text not null,
  headcount integer not null,
  start_date date,
  housing_provided text,
  contact_name text not null,
  phone text not null,
  notes text,
  status text default 'new'
);
```

### 6. API Routes

`/app/api/submit-candidate/route.ts` — принимает данные анкеты + теста, сохраняет в Supabase.
`/app/api/submit-employer/route.ts` — принимает форму работодателя, сохраняет в Supabase.

В обоих — после сохранения опционально стрельнуть в `N8N_WEBHOOK_URL` из `.env` (если переменная задана).

## Важные детали

1. **Анимации** — при смене шага анкеты плавный fadeUp (CSS transition, не библиотека)
2. **Кнопка "Назад"** на каждом шаге анкеты
3. **Валидация** — обязательные поля помечены *, при пустых — красная рамка
4. **Мобильная адаптация** — сайт должен хорошо работать на телефоне (много пользователей придут с мобилки через Telegram)
5. **Язык** — весь интерфейс на русском
6. **Не нужно делать** — авторизацию, дашборд менеджера, панель администратора (это следующий этап)
